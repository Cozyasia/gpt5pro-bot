# v70 vocalclip flow + Telegram file fallback

Что исправлено поверх v69:

1. Режим `🎤 Клип с вокалом (1 человек)` теперь не теряет сценарий после загрузки фото.
   - Если пользователь сначала выбрал режим, а потом загрузил портрет, бот сразу просит описание песни/клипа.
   - Больше не должен возвращаться в общее меню `Фото получено. Что сделать?`.

2. В общее меню действий после загрузки фото добавлена кнопка:
   `🎤 Клип с вокалом (1 человек)`.

3. Добавлена обработка `pedit:vocalclip`:
   - если фото уже загружено, можно прямо из меню фото запустить сценарий клипа с вокалом.

4. Улучшена отправка результата Kling Avatar / видео:
   - если Telegram не принимает оригинальный MP4 напрямую, бот пробует сжать MP4 локально через ffmpeg;
   - только если сжатая версия тоже не отправилась, оставляет ссылку.

Новые ENV по желанию:

```env
TELEGRAM_RESULT_MAX_MB=48
TELEGRAM_VIDEO_COMPRESS_ON_FAIL=1
```

Оставить из v69:

```env
VOCAL_CLIP_MAX_AUDIO_S=65
VOCAL_CLIP_MIN_AUDIO_S=12
VOCAL_CLIP_KLING_MAX_WAIT_S=1800
```

Проверка после деплоя:

```text
/version
```

