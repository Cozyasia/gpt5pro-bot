# Neyro-Bot v85 — Presentation & Catalog Studio

Версия: `v85-presentation-studio-2026-07-13`

## Что добавлено

В раздел `💼 Работа/Бизнес` встроен полноценный мастер:

- `📊 Презентация PDF + PPTX`
- `📕 Каталог PDF + PPTX`

Пользователь проходит последовательный сценарий:

1. Отправляет бриф текстом, голосом или документом.
2. Получает структуру 7–14 слайдов и утверждает либо исправляет её.
3. Выбирает работу с логотипом:
   - загрузить свой PNG/JPG;
   - создать 3 варианта по брифу и выбрать один;
   - продолжить без логотипа.
4. Выбирает изображения:
   - массовая загрузка своих фото;
   - AI-генерация;
   - смешанный режим;
   - проект без фото.
5. Загружает фотографии по одной, альбомом или ZIP-архивом.
6. Выбирает стиль и цветовую палитру.
7. При необходимости создаёт недостающие визуалы через Midjourney или OpenAI/Comet Images.
8. Подтверждает проект.
9. Получает два согласованных файла:
   - редактируемый `PPTX`;
   - готовый для отправки/печати `PDF`.

Проекты, состояния и пути к ресурсам сохраняются в SQLite. Фотографии и готовые файлы хранятся на диске Render в `/data/presentation_studio`.

## Файлы для репозитория

Замените существующие файлы:

```text
main.py
engine.py
render.yaml
env.sample
premium.html
requirements.txt
runtime.txt
README.md
```

Добавьте новый файл:

```text
presentation_studio.py
```

Не удаляйте Render Disk и файл `/data/subs.db`.

## Новые ENV

В `render.yaml` значения уже добавлены. При ручном добавлении используйте:

```env
PRESENTATION_DATA_DIR=/data/presentation_studio
PRESENTATION_MAX_UPLOADS=60
PRESENTATION_MAX_GENERATED_IMAGES=8
PRESENTATION_RENDER_COST_USD=0.20
PRESENTATION_IMAGE_ENGINE_AUTO=1
```

Остальные действующие ключи и переменные изменять не требуется.

`PRESENTATION_RENDER_COST_USD` — внутренняя стоимость финальной сборки до общей розничной наценки `GENERATION_PRICE_MULTIPLIER`. Кредиты окончательно списываются только после успешной отправки обоих файлов.

AI-логотипы и AI-изображения используют существующие ключи:

```text
OPENAI_IMAGE_KEY / OPENAI_API_KEY
COMET_API_KEY
MIDJOURNEY_ENABLED
```

## Деплой

1. Сделайте резервную копию `/data/subs.db`.
2. Загрузите все файлы v85 в репозиторий.
3. Убедитесь, что `presentation_studio.py` находится рядом с `main.py`.
4. В Render выполните `Manual Deploy` → `Clear build cache & deploy`.
5. Дождитесь строки запуска с версией:

```text
v85-presentation-studio-2026-07-13
```

## Проверка

В Telegram выполните:

```text
/version
/diag_presentation
```

Затем:

1. `💼 Работа/Бизнес` → `📊 Презентация PDF + PPTX`.
2. Отправьте тестовый бриф.
3. Утвердите структуру.
4. Создайте 3 логотипа или загрузите свой.
5. Загрузите несколько фото одним альбомом.
6. Проверьте ZIP с JPG/PNG.
7. Выберите стиль и палитру.
8. Соберите проект.
9. Убедитесь, что бот прислал и `.pptx`, и `.pdf`.

## Команды

```text
/presentation — начать презентацию
/catalog — начать каталог
/diag_presentation — диагностика мастерской
```

## Архитектура

Новые таблицы создаются автоматически:

```text
presentation_projects
presentation_assets
```

Состояния проекта переживают перезапуск приложения. Пользователь может продолжить незавершённый проект после нового деплоя.

PDF и PPTX рендерятся из одного утверждённого JSON-макета. Поэтому структура, изображения, логотип, стиль и палитра совпадают в обоих форматах.
