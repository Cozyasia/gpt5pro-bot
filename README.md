# v54-full-v35.6-plus-avatar-photoclip-menu-2026-07-09

Исправление меню после v53.

Причина: логика Kling Avatar и Photo→videoclip была добавлена в код и в кнопки после загрузки фото, но основное меню «🔥 Развлечения», которое открывается через ReplyKeyboard/режимы, строилось функцией `_mode_kb("fun")`. В ней кнопки `🗣 Говорящий аватар` и `🎵 Фото → видеоклип` не были добавлены, поэтому в Telegram они не отображались.

Что исправлено:

- В главное подменю «🔥 Развлечения» добавлены кнопки:
  - `🗣 Говорящий аватар`
  - `🎵 Фото → видеоклип`
- Для говорящего аватара добавлено подменю:
  - загрузить портрет;
  - использовать последнее фото;
  - текст → говорящий аватар;
  - voice/audio → говорящий аватар.
- Для фото→видеоклип добавлено подменю:
  - загрузить фото;
  - использовать последнее фото;
  - кинематографичный клип;
  - Reels/Shorts 9:16;
  - travel/luxury клип;
  - музыкальный клип;
  - рекламный клип;
  - свой промпт.
- Если выбран пресет фото→видеоклип и фото ещё нет, бот запомнит пресет и автоматически запустит генерацию после загрузки фото.
- Обновлена версия `/version`: `v54-full-v35.6-plus-avatar-photoclip-menu-2026-07-09`.
- `python -m py_compile main.py` пройден успешно.

ENV менять не нужно, если уже добавлен блок из v53 для Kling Avatar / Photo→videoclip.
