# Neyro-Bot v136 — Celebrity Selfie quality + GPT/Gemini chats

Release: `v136-celebrity-selfie-chat-choice-2026-07-20`

## Celebrity Selfie

The production route keeps the audited single-owner Telegram wizard and changes the quality pipeline:

1. The user may upload one selfie or add a second angle.
2. Up to eight labelled identity references are sent to the image model.
3. Two independent Gemini Pro Image candidates and one Gemini Flash Image candidate are generated by default.
4. Every candidate must be one continuous frame with exactly two foreground adults and no collage/split screen.
5. User identity and public-person identity are scored separately.
6. The official OpenAI Images edit endpoint is used as a high-fidelity independent fallback.
7. When only one face is weak, only that face is repaired; the already-good face and the scene are preserved.
8. FLUX.2 Pro/Max is an optional provider when `BFL_API_KEY` exists.
9. The normal route no longer applies PiAPI multi-face-swap to both faces.
10. Aspect ratio is selected from the scene: 4:5 by default, 3:2 for wide/event scenes, 1:1 for close portraits, 9:16 for stories.

Safe defaults are set in `neyrobot_prod/__init__.py`; no Render Environment change is required for Gemini or OpenAI because existing keys are reused. FLUX remains optional until `BFL_API_KEY` is configured.

## Selectable chats

A new **💬 Мои чаты** button and `/chat` command open two persistent choices:

- **Чат с GPT** — the existing official GPT-5-family router;
- **Чат с Gemini** — Gemini `gemini-3.5-flash`, with `gemini-3.1-flash-lite` fallback.

The choice is stored per Telegram user/chat and, when the existing virtual-conversation identifier is available, per virtual conversation. GPT and Gemini histories are persisted separately. If Gemini is temporarily unavailable, the request falls back to GPT without losing the selected preference.

## Diagnostics after Render deploy

- `/version` → `v136-celebrity-selfie-chat-choice-2026-07-20`
- `/diag_chat` → selected provider, Gemini key/model state and last route
- `/diag_celebrity_flow` → candidate counts, individual identity scores, selected provider, targeted repair and optional FLUX state
- `/diag_text` → official GPT router state

## Manual acceptance test

1. Open **Развлечения → Селфи со звездой**.
2. Upload a clear front selfie and add a second slight-angle photo.
3. Choose a catalog person and the **Премьера** scene.
4. Confirm that one continuous image is returned, both identities are distinct, and no source-sheet/collage appears.
5. Repeat with **Яхта/съёмочная площадка** and verify the automatic wider aspect.
6. Open **💬 Мои чаты**, choose Gemini, ask two follow-up questions, switch to GPT, and verify the GPT conversation does not inherit the Gemini history.
