# Neyro-Bot v86 — logo stability, text-safe visuals, Runway fallback

Версия: `v86-logo-runway-resilience-2026-07-13`

## Что исправлено

### 1. Логотипы в мастере презентаций

После утверждения структуры бот спрашивает только, есть ли готовый логотип:

- загрузить свой файл;
- создать 3 варианта автоматически;
- продолжить без логотипа.

Повторный бриф больше не требуется. Название бренда, сфера, аудитория, стиль и назначение берутся из уже утверждённого проекта.

Добавлены:

- защита от двойного нажатия и параллельного запуска;
- состояние `generating_logo`;
- ровно три варианта даже при частичном ответе провайдера;
- локальные резервные варианты;
- fallback `photo → document`, если Telegram не принимает карточку;
- изоляция ошибок одного варианта, чтобы остальные продолжали отправляться;
- кнопки повторной генерации и необязательного уточнения;
- корректное извлечение короткого названия бренда вместо полного заголовка презентации.

Генерация вариантов логотипа включена в мастер презентации и не создаёт отдельное ошибочное списание при недоступности image-провайдера.

### 2. Midjourney и нечитаемый текст на этикетках

Для бутылок, упаковки, вывесок, экранов, меню и других объектов с надписями действует text-safe режим:

- изображение создаётся с пустой этикеткой или пустым полем;
- запрещаются псевдобуквы, глифы и случайные числа;
- точное название бренда, цены и тексты добавляются кодом при сборке PPTX/PDF;
- text-sensitive визуалы автоматически направляются в OpenAI/Comet Images, даже если пользователь выбрал Midjourney;
- Midjourney остаётся для атмосферных, lifestyle и luxury-сцен без обязательного текста.

На product-слайдах добавляется читаемая брендовая плашка поверх макета.

### 3. Runway text→video

Исправлена production-логика:

1. Если указан `RUNWAY_API_KEY`, первым используется официальный Runway API.
2. Для text→video применяется `gen4.5` и `POST /v1/image_to_video` без `promptImage`.
3. Если официального ключа нет, бот пробует совместимый канал Comet.
4. Ответы Comet `503`, `model_not_found`, `no available channel` не показываются пользователю как длинный JSON.
5. Недоступный Comet-канал ставится на cooldown.
6. При недоступности Runway автоматически запускается Kling без дополнительного списания.
7. Если и резерв не сработал, пользователь получает короткое понятное сообщение, а кредиты освобождаются.

## Деплой

1. Сохраните резервную копию `/data/subs.db`.
2. Замените все файлы из архива в репозитории.
3. Убедитесь, что `presentation_studio.py` находится рядом с `main.py`.
4. Выполните `Manual Deploy → Clear build cache & deploy`.
5. После запуска проверьте:

```text
/version
/diag_presentation
/diag_runway
```

Ожидаемая версия:

```text
v86-logo-runway-resilience-2026-07-13
```

## Новые ENV

Значения уже внесены в `render.yaml` там, где секрет не требуется:

```env
RUNWAY_DIRECT_FIRST=1
RUNWAY_TEXT_MODEL=gen4.5
RUNWAY_COMET_TEXT_MODELS=runway-video,gen4.5
RUNWAY_TEXT_FALLBACK_KLING=1
RUNWAY_HIDE_TECH_ERRORS=1
PRESENTATION_TEXT_SAFE_VISUALS=1
PRESENTATION_FORCE_OPENAI_FOR_TEXT=1
```

Официальный ключ Runway опционален и добавляется вручную в Render:

```env
RUNWAY_API_KEY=ваш_официальный_ключ_Runway
```

Без него бот использует Comet и автоматически переключается на Kling, если Runway-канал отсутствует.

## Ручной тест

### Логотип

1. Создать каталог.
2. Утвердить структуру.
3. Нажать `Нет — создать 3 варианта автоматически`.
4. Проверить, что бот не просит новый бриф.
5. Проверить, что появились 3 варианта и нет сообщения `Упс`.
6. Быстро нажать кнопку генерации дважды: второй запуск должен сообщить, что задача уже выполняется.
7. Выбрать вариант и продолжить проект.

### Text-safe изображения

1. Создать каталог бутылок/упаковки.
2. Выбрать Midjourney.
3. Проверить, что для text-sensitive кадров бот использует безопасную маршрутизацию и пустые этикетки.
4. Проверить читаемые названия и цены в готовом PDF/PPTX.

### Runway

1. Отправить голосом запрос на text→video.
2. Выбрать Runway.
3. При отсутствии Comet-канала бот должен коротко сообщить о переключении.
4. Должен автоматически запуститься Kling.
5. Длинный JSON `model_not_found/no available channel` пользователю не показывается.
